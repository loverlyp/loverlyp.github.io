指令(directive)

指令是在 html 标签中的一个有特殊函义的属性，其属性值为 JS 表达式。

  • v-html：渲染 html 文本

  • v-text：渲染纯文本

  • v-bind：动态绑定属性，可简写为 :

  • v-on：绑定事件处理程序，可简写为 @

  • v-for：列表渲染，用于对数组或对象属性实现循环遍历（迭代）

    列表渲染时，如果调用数组的变异方法（变更方法：调用这些方法后会改变原数组本身）修改数组元素，则页面会响应式更新，数组有7个变异方法：

    • push()/pop()
    • unshift()/shift()
    • splice()
    • sort()/reverse()

    如果调用的是非变异方法（不影响原数组本身的方法，如：slice()、filter()、concat()），又想将其返回的新数组在页面中响应式渲染，则可以使用替换数组的方式，即使用赋值运算符 = 重新为数组变量赋值。

  • v-if、v-else-if、v-else：条件渲染

    v-if 满足条件的元素会被显示，不满足条件的元素会被隐藏，v-if 操作的是 DOM 树节点的添加或销毁

  • v-show：条件渲染

    v-show 满足条件的元素会被显示，不满足条件的元素会被隐藏，v-show 操作的是节点的 CSS 样式中的 display（隐藏则为 display: none，显示则去掉 display 属性设置）

    v-if 与 v-show 的区别：

    v-if 有更高的切换开销，而 v-show 有更高的初始渲染开销。因此，如果需要非常频繁地切换，则使用 v-show 较好；如果在运行时条件很少改变，则使用 v-if 较好。

    v-if 与 v-for 同时使用：

    在 vue2.x 版本中，在同一个节点中同时使用 v-if 与 v-for 指令，v-for 优选级更高。不推荐在同一个节点中同时使用这两个指令。

  • v-model：双向数据绑定，通常用于表单元素值的双向绑定。

    你可以用 v-model 指令在表单 <input>、<textarea> 及 <select> 元素上创建双向数据绑定。

    v-model 实际上是一个语法糖，在内部为不同的输入元素使用不同的 property 并抛出不同的事件：

    • text 和 textarea 元素使用 value property 和 input 事件；
    • checkbox 和 radio 使用 checked property 和 change 事件；
    • select 字段将 value 作为 prop 并将 change 作为事件。

  • v-slot：在 2.6.0 中新增的指令，用于父组件向子组件的插槽中分发内容，可简写为 #（v-slot 需要使用在组件或 <template> 标签内）

    组件

    组件系统是 Vue 的另一个重要概念，因为它是一种抽象，允许我们使用小型、独立和通常可复用的组件构建大型应用。

    定义组件

    创建组件选项对象：

    template: 用于描述组件对应的视图布局结构，需要注意的是，如果布局结构中有多个元素节点，必须使用单个根元素节点包裹所有布局结构。

    （面试）为什么组件中的 data 要定义成函数？

    组件是可能被复用的（即根据一个组件结构，创建多个组件实例），如果 data 是普通对象，则多个复用的组件实例使用到的数据是指向同一个 data 对象，当其中某一个组件实例修改了数据后，其它组件实例也会受到影响，这与实际业务不符，所以应该每个组件实例都有自己独立的数据结构。将 data 设计为函数，则创建组件实例时，每创建一个组件实例，调用一次 data() 函数，在函数体内部返回一个新的对象（内存中会重新开辟空间保存新对象的数据），这样，多个组件实例间数据就是相互独立的，一个组件实例数据的修改不会影响到其它组件实例。

组件通信

父子组件

  • 父传子：使用 props （属性）。在父组件中使用子组件时，通常通过动态绑定属性值向子组件传递数据，在子组件定义时，利用 props 字段声明其可接收到的属性。在 props 中声明的属性也会被自动挂载到组件实例下。
  • 子传父：利用事件来传递数据。在父组件中使用子组件时，利用 v-on 绑定一个自定义事件（事件处理程序会使用到某些参数值来修改父组件中的数据），在子组件中需要传递数据时，触发在父组件中绑定的对应事件即可（利用 $emit(事件名称, 传递的数据) 方法来触发事件的执行）

跨组件跨层级

  • 转换为父子关系
  • event-bus：事件总线（不需要在每一层的组件中都添加事件机制来处理数据的传递）。利用的是 Vue 组件实例的事件特性（$on 用于注册事件监听，$emit 用于触发执行事件处理程序）。创建一个全局的空的 Vue 实例，在需要接收数据的组件中，调用 $on() 注册事件监听，在需要传递数据的组件中，调用 $emit() 触发事件（触发事件执行的同时，传递数据）。
  • vuex：状态管理

计算属性和侦听器

计算属性是可以缓存的，计算属性是基于它们的响应式依赖进行缓存的，只在相关响应式依赖发生改变时它们才会重新求值。

计算属性（computed） VS 方法（methods）：

  • 计算属性有缓存，方法没有缓存（计算属性其依赖项不改变时，不会重新求值，而方法每调用一次都会重新执行方法主体）

计算属性（computed） VS 侦听属性（watch）：

  • 计算属性有缓存，侦听属性没有缓存
  • 计算属性通常是由一个或多个依赖计算出一个新的值，而侦听属性通常是由一个数据的变化导致其它一个或多个数据的变化
  • 计算属性中通常不包含副作用操作（比如网络请求），而侦听属性中可包含副作用操作

组件生命周期

生命周期：从组件创建开始，到组件销毁，所经历的整个过程。

生命周期钩子函数：是在生命周期过程中会被自动调用执行的函数，在这些函数中，我们可以添加一些额外需要执行的任务。

  • create 阶段
    • beforeCreate()
    • created()：实例创建后，这时，可以在 vue 实例中调用到 data 中设置的数据了。可以在这个生命周期钩子函数中发起网络请求。
  • mount 阶段
    • beforeMount()
    • mounted()：在挂载结束后执行，这时，可以获取到渲染后的实体 DOM 元素了。
  • update 阶段
    • beforeUpdate()
    • updated()
  • destroy 阶段：
    • 通常会销毁的资源有：定时器、之前注册的事件监听、未完成的异步请求、websocket……
    • beforeDestroy()
    • destroyed()

Promise

文档

三种状态：

  • pending：待定、等待
  • fulfilled：兑现，成功
  • rejected：拒绝，失败

它让您能够把异步操作最终的成功返回值或者失败原因和相应的处理程序关联起来。 这样使得异步方法可以像同步方法那样返回值：异步方法并不会立即返回最终的值，而是会返回一个 promise（代表未来执行结构的对象），以便在未来某个时候把值交给使用者。

创建 Promise 对象:

  • resolve：函数结构，将 Promise 状态修改为 fulfilled
  • reject：函数结构，将 Promise 状态修改为 rejected

Promise.prototype.then(onFulfilled[, onRejected])

是前一个 Promise 导步任务执行结束，接下来需要继续执行的任务。
